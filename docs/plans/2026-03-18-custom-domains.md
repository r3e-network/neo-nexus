# Custom Endpoint Domains Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Allow users to specify their own "Vanity Domain" (e.g., `rpc.mycompany.com`) for their dedicated nodes, similar to the functionality provided by Chainstack or QuickNode.

**Architecture:**
1. **Extend Schema:** Add a `customDomain` string field to the `Endpoint` model in Prisma.
2. **Update Addressing Service:** Modify `EndpointAddressing.ts` to prioritize `customDomain` over the auto-generated `node-region-id.neonexus.cloud` address if one is configured.
3. **UI Implementation:** In `EndpointDetailsClient.tsx`, add a new configuration form in the "Settings" tab allowing the user to input a custom domain name. Provide a visual CNAME target they need to map their DNS to.
4. **Action Integration:** Create a Server Action to update the custom domain and regenerate the endpoint URL, then trigger the Apisix Route re-sync.

**Tech Stack:** Next.js App Router, Prisma, TypeScript.

**Files:**

**Step 1:**

### Task 2: Update Addressing Engine

**Step 1: Write failing tests**

### Task 3: Expose Custom Domain UI

**Files:**
- Modify: `dashboard/src/app/app/endpoints/settingsActions.ts`
- Modify: `dashboard/src/app/app/endpoints/[id]/EndpointDetailsClient.tsx`
- Modify: `dashboard/src/app/app/endpoints/[id]/page.tsx`

**Step 1: Implement Action**
- Create `updateCustomDomainAction(endpointId, customDomain)` in `settingsActions.ts`.
- It should update the DB, recalculate the URLs via `EndpointAddressing`, save them, and call `syncEndpointToApisix` to update the routing layer.

**Step 2: Update UI**
- Ensure `customDomain` is selected in `page.tsx` and passed to the client.
- In `EndpointDetailsClient.tsx`, under the "Settings" tab, add a "Vanity Domain" section.
- If they enter `rpc.mycompany.com`, show a helper text: *"Create a CNAME record in your DNS provider pointing `rpc.mycompany.com` to `{autoGeneratedDomain}`."*
